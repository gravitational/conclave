Generate {{.N}} identical prompts for independent security reviews. Each prompt MUST include the following severity definitions and instructions verbatim:

The agent should ONLY report Critical or High severity findings. Do NOT report Medium, Low, or Informational issues.

**Critical** — An attacker can achieve ANY of: remote code execution, authentication bypass granting access to any account, full database compromise, complete privilege escalation to admin/root, or unrestricted file read/write on the server.

**High** — An attacker can achieve ANY of: stored XSS executing in other users' sessions, IDOR/broken access control exposing other users' sensitive data, SSRF reaching internal services, credential leakage (hardcoded secrets, tokens in logs, exposed API keys), insecure deserialization with a viable gadget chain, or second-order injection with a realistic trigger.

**NOT Critical or High** (do not report): self-XSS, reflected XSS requiring unlikely interaction, missing rate limiting/CSRF/headers, DoS without data exposure, information disclosure of non-sensitive data, theoretical vulnerabilities requiring unlikely access, best-practice violations, missing input validation without a demonstrated exploit path.

Each prompt should instruct the agent to pay special attention to parallel or duplicate implementations of the same logic, as these are a rich source of exploitable bugs. This includes: duplicate validation/authorization paths where one path may miss constraints the other enforces, client-side vs server-side logic where the server may be weaker, multiple API endpoints for the same operation where one is less hardened, redundant data processing pipelines with differences in sanitization, and replicated state machines where inconsistencies allow illegal state transitions. When parallel implementations are found, the agent should diff them carefully — the vulnerability is in the gap between them.

Each prompt should instruct the agent to identify the SINGLE MOST CRITICAL security vulnerability in this subsystem. The agent must provide: (1) specific vulnerable code location, (2) clear attack vector explanation, (3) a concrete step-by-step exploitation scenario, (4) what the attacker gains, and (5) severity rating (Critical or High only) with justification referencing the definitions. If multiple Critical/High issues exist, report ONLY the most severe one. If no Critical or High vulnerabilities are found, say "No critical vulnerabilities found" — this is a valid outcome; do not stretch to report lower-severity issues.
