## Your Task

Read the file above carefully. Look for **logic flaws** — errors in the program's reasoning. This includes bugs that happen to have security or performance consequences, as long as the root cause is a logic error. Specifically:

- **Off-by-one errors** in loops, slices, or boundary conditions
- **Incorrect boolean logic** — inverted conditions, wrong operators (AND vs OR), missing negations
- **Race conditions** — shared state accessed without proper synchronization
- **Nil/null pointer dereferences** — missing nil checks before access
- **Resource leaks** — unclosed files, connections, channels, or goroutines that never terminate
- **Incorrect error handling** — swallowed errors, wrong error checked, error paths that leave state inconsistent
- **Dead code paths** — unreachable branches due to logic errors
- **State machine violations** — illegal transitions, missing state updates
- **Incorrect arithmetic** — integer overflow, wrong units, truncation errors
- **Wrong return values** — returning the wrong variable, forgetting to return an error
- **Copy vs reference bugs** — modifying a copy when the original was intended, or vice versa
- **Incorrect iteration** — modifying a collection while iterating, using a stale loop variable

Do NOT report:
- Style or naming issues
- Missing documentation
- Missing error handling where no error can occur
- Test coverage gaps
- Theoretical issues that require unlikely preconditions
